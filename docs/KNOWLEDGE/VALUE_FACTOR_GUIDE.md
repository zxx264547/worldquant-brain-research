# ValueFactor (VF) 攻略

## 核心概念

- **VF**：决定你能吃多大碗饭，是保底关键因素
- **OS（Osmosis Rank）**：隐藏大佬，一周更新5次，跟Base强绑定
- **OS 0.5是分水岭**

## 高效策略

| OS状态 | 策略 | 效果 |
|--------|------|------|
| OS < 0.5 | 去交其他区域Alpha稳住收益 | Theme Buff基本等于白给 |
| OS > 0.5 | 交Theme | 高OS + Theme加成，收益翻倍 |

## 不需要交三五也可以拿高Base

OS很高时，不需要交三五就可以拿50+的Base

## 四大KPI

| KPI | 说明 | 重要性 |
|-----|------|---------|
| **VF** | 决定Base大小 | 核心 |
| **OS** | 隐藏加分项 | 关键 |
| **Alpha质量** | 1个fit2的ra > 4个PPA | 效率 |
| **Datasets Theme** | 两周换一次，上榜有Buff | 倍增 |

## VF提升经验

1. 坚持提交，保持提交数量
2. 关注OS排名，冲上0.5以上再交Theme
3. Combine稳定，避免负分
4. 低相关性 + 高fit才是硬通货
