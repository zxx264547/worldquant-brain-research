## 完整研究流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Alpha研究工作流                            │
└─────────────────────────────────────────────────────────────┘

1. 数据准备
   ├─ 下载数据集到 data/datasets/
   ├─ 运行 webdata_raw_parser.py 解压
   └─ 分析字段命名规律

2. Alpha挖掘
   ├─ batch_mining.py 启动挖掘
   ├─ 设置筛选参数
   └─ 并发控制(10并发)

3. 多轮筛选
   ├─ screening_pipeline.py
   ├─ PPA标准: PPC<0.5, Sharpe>1.0, Fitness>0.5
   └─ Margin > Turnover

4. 相关性去重
   ├─ correlation_analysis.py
   ├─ 分族 + 组内排序
   └─ 每族保留1-2条代表

5. 续航力评分
   ├─ os_score_calculator.py
   ├─ D1-D4四维度
   └─ 75+ STRONG BUY

6. 提交前检查
   ├─ 行业中性/市场中性
   ├─ 年度收益均匀性
   └─ 确认无相似因子

7. 提交入库
   ├─ 记录到 factor_library.db
   ├─ 保存表达式到 factors/
   └─ 更新每日日志
```

### 每日工作流程

| 时间 | 任务 |
|------|------|
| 早间 | 查看OS排名和前日表现 |
| 日间 | 执行Alpha挖掘任务 |
| 晚间 | 整理当日结果，更新日志 |

### 参数配置

| 参数 | 值 | 说明 |
|------|-----|------|
| min_sharpe_step1 | 0.7 | 首轮 |
| min_sharpe_step2 | 0.7 | 次轮 |
| min_sharpe_step3 | 1.0 | 第三轮 |
| min_fitness | 0.5 | Fitness门槛 |
| max_turnover | 0.7 | 最大换手率 |
| correlation_threshold | 0.8 | 相关性阈值 |
| batch_size | 10 | 批大小 |
| max_variants | 20 | 最大变体数 |
| concurrent_workers | 10 | 并发数 |

### 变体生成策略

1. **参数变化**: 调整时间窗口参数
2. **算子组合**: 尝试不同算子组合
3. **数据字段**: 同一字段的不同表达
4. **中性化**: 添加/移除中性化

## 提交前检查清单

### 核心指标检查

| 检查项 | 标准 | 权重 |
|--------|------|------|
| PPC | < 0.5 | 核心 |
| Sharpe | >= 1.0 | 核心 |
| Fitness | > 0.5 | 核心 |
| Margin | > Turnover | 核心 |

### 质量检查

| 检查项 | 说明 |
|--------|------|
| [ ] 中性化 | 必须做行业中性/市场中性 |
| [ ] 年度均匀 | 年度收益不要极端依赖某几年 |
| [ ] 相关性 | 避免与已有因子高度相关 |
| [ ] 稳定性 | 实盘Sharpe不能显著低于样本内 |

### OS策略

| OS状态 | 建议策略 |
|--------|----------|
| OS < 0.5 | 交其他区域Alpha，不交Theme |
| OS >= 0.5 | 交Theme，加成翻倍 |
